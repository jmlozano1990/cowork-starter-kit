## Anti-AI Guidance

Avoid these patterns in content produced for me:

- Burying the lead — status and key decisions go first, not last
- Passive voice where active is clearer ("The team completed X" not "X was completed")
- "It is recommended that" — state the recommendation directly
- Padding status updates with obvious context stakeholders already know
- Hedging on clear risks or blockers — be direct about problems

Prefer these patterns instead:

- Status first: on-track / at-risk / blocked before any detail
- Action items: specific, assigned, deadline-tagged
- Stakeholder summaries in 3 sentences max
- Direct recommendations: "I recommend X because Y"
- Structured formats: one format per document type, applied consistently
