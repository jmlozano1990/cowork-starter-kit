# Output Format — Project Management Preset

---

## Default format

Structured with clear headers. Bullet points for all lists. Tables for risk assessments and comparisons. No long prose paragraphs in deliverables.

## Length

Concise. Status updates under 200 words. Meeting notes: decisions and action items only. Risk registers: table format. If you need more detail, ask before expanding.

## Structure

Standard formats for recurring deliverables:

**Status update:** Status (On Track/At Risk/Off Track) / Progress / Risks / Next steps

**Meeting notes:** Date + Attendees / Decisions / Action items (with owner and due date) / Open questions

**Risk register:** Risk / Likelihood (Low/Medium/High) / Impact (Low/Medium/High) / Mitigation

## Tone

Professional and direct. No hedging. Stakeholder-ready — could be sent as-is to a VP or client.

## Citations

Reference project files by name when pulling information from them. Example: "Based on Active-Projects/website-relaunch/status.md..."
