### 4. Final Recommendation for v2.2 Source

**Recommendation: Path C — ship architecture-only, defer first external source to v2.3.**

Rationale:

The re-scan found one new candidate that clears the vetting bar (`contract-review` from `evolsb/claude-legal-skill`), but its persona coverage is narrow (Riley primary, Maria secondary at low frequency). Two of six personas at moderate-to-low frequency does not meet the bar for a first external source that sets the ADR-028 implementation precedent.

The original recommended candidate (Weizhena) fails the best-in-class filter. The highest-quality remaining candidate (ComposioHQ `meeting-insights-analyzer`) is a strong gap-fill but requires significant adapter work that is out of scope for v2.2's architecture cycle.

Path C is the right call for three reasons:

1. **ADR-028 is the real v2.2 deliverable.** The pinned-digest architecture (content_sha256 field in lock schema, second source in .cowork-allowlist.json) is the infrastructure that makes all future external sources trustworthy. Shipping ADR-028 implementation cleanly — without forcing a mediocre first source — is more durable than wiring in a source that doesn't fully meet the bar.

2. **v2.3 has two strong candidates.** `evolsb/contract-review` (narrow but high-quality) and `ComposioHQ/meeting-insights-analyzer` (broader, higher adapter cost) are both ready for a v2.3 evaluation once the adapter framework and ADR-028 are implemented. The user can choose between a narrow/deep pick vs. a broad/medium pick at that point.

3. **No forced source pick is better than a forced mediocre one.** The user explicitly authorized this outcome.

If the user wants to de-risk v2.3, one optional pre-work action is available now: file a GitHub issue on `lyndonkl/claude` asking the maintainer to add an MIT license. If they do, `decision-matrix` (5-persona coverage, strong content) becomes a top-tier v2.3 candidate with no adapter work needed beyond the `trigger_examples` field.
