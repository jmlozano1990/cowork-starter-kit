| ADR-spec drift on parameterized artifacts | WARNING | v1.2, v2.0 (C8/B2/A3), v2.0.2 (S1) | When a spec, ADR, or release artifact describes a parameterized list (category count, feature checklist, CHANGELOG entry), implementations ship with placeholder values or subsets instead of the final computed value. Mitigation strengthened (v2.0.x): byte-comparison of frozen list text against live source file required at Phase 5, not cardinality check alone. | 2026-05-07 |
| Cumulative-feature shipping with external-trigger workflow gating produces post-merge layer-onions | WARNING | v2.0.0 (1-cycle, watch v2.1+) | When a feature includes a new long-running workflow (cron + workflow_dispatch + multi-step), each post-merge BLOCKER masks the next layer because all prior layers must be fixed before the next one is visible. v2.0.x produced a 5-layer onion: YAML structure → hallucinated SHA → missing auth → repo permission → subshell scope. Mitigation: any cycle adding a new external-trigger workflow MUST include a dry-run job at the same PR exercising ≥2 representative e2e paths before Phase 7 APPROVED. | 2026-05-06 |
| Git-State Divergence — Cycle Authored on Stale Base | WARNING | v2.2 (1-cycle, RECURRENCE-MITIGATED-BY-PROPOSED-GUARD) | When parallel Claude Code sessions write to local main while origin/main advances through PR merges, a divergence accumulates silently. A new cycle authored on the stale local base produces Phase 0 and Phase 1 artifacts whose underlying assumptions (VERSION, file line counts, target block existence) are incorrect for the origin state. In v2.2: local main was behind 2 commits (v2.0.5, v2.1.0 tag), stale base caused Phase 1 to flag "v2.1 has not shipped" as a sequencing precondition — saved by @architect's organic discipline (cat VERSION + wc -l WIZARD.md). Without that check, @dev would have patched a non-existent line at Phase 4. Proposed mitigation: check-base-sync.sh pre-/spec guard in The-Council scripts/ (sibling to check-stale-cycle.sh) that git-fetches and blocks if local branch is behind origin. | 2026-05-08 |
| Recurring Version Artifact Miss | WARNING | v2.1, v2.2 (RESOLVED in v2.3.0 via explicit constraint enumeration; v2.3.1 CONFIRMS RESOLUTION HOLDS — 2nd consecutive cycle all 4 release artifacts shipped correctly) | Two consecutive cycles shipped without the README badge and "Next up" teaser (part of the standard release artifact checklist). General reminders in Phase 4 constraints failed to prevent the miss. Resolution: bind all 4 release-artifact sub-items explicitly as separate verifiable items in the Phase 4 constraint (VERSION, CHANGELOG, README badge with exact badge string, Next-up teaser with exact version reference). This is the canonical precedent for fixing recurring artifact misses through constraint disaggregation rather than general reminders. v2.3.1 AC-REL-1..4 all PASS on first push — resolution confirmed stable for 2 consecutive cycles. | 2026-05-08 |
| Local Lint vs CI Divergence | WARNING | v2.3.0 (1-cycle, WATCH); v2.3.1 result: 0 MD058 failures, WATCH continues | Cowork's CI runs markdownlint-cli2 on all markdown files, but the pipeline has no equivalent local lint step @dev can run before push. Any annotation or formatting strategy specified by @architect at Phase 1 can only be validated by CI, creating a guaranteed rework loop when format violates a lint rule. First manifestation: MD058/blanks-around-tables on curated-skills-registry.md (v2.3.0 Phase 5, pre-merge). Root cause is structural (tooling gap), not agent-quality. v2.3.1 had 0 CI lint failures (no table-adjacent content in 8 SKILL.md files). Pattern stays WATCH — structural tooling gap not yet closed (CF-4 local markdownlint pre-commit still deferred). Proposed mitigation: add local markdownlint-cli2 step to cowork workflow for @dev to self-check before push. | 2026-05-08 |
| Paperwork-Follow-Up-PR-Pattern | WARNING | v2.3.0 (first instance — orphan docs swept up by v2.3.1 PR #39), v2.3.1 (second instance — separate PR #39 for 10 docs files), v2.4 (third instance — PR #42 for Phase 5/6 reports + public-artifact refresh; F7 resolved Phase 0/1/2 gap but temporal gap for Phase 5/6 docs persists) (WATCH — 3-cycle CONFIRMED, promoted) | Code PR ships clean to main; docs artifacts authored during the cycle are not committed on the release branch before merge. Direct-to-main push is blocked by harness permission gate (correct enforcement). A separate paperwork PR is required post-merge. Root cause now refined in two layers: (1) Phase 0/1/2 paperwork — resolved by v2.4 F7 mandatory-commit topology (Commit 6 REQUIRED). (2) Phase 5/6 paperwork — structurally incompatible with the Phase 4 commit topology because qa-report and security-audit do not exist when Phase 4 commits are authored. F7-temporal-gap sub-pattern: first instance, WATCH 1/3. Proposed next mitigation: 9-commit topology with post-Phase-6 Commit 8/9 slot, or paperwork-branch auto-creation at Phase 7 close. Read by @architect at Phase 1 and @dev at Phase 4. Council self-improve candidate: C2 F7-temporal-gap fix. | 2026-05-09 |
| Public-Artifact-Hygiene-Gap | WARNING | v2.4 (first instance — README, SETUP-CHECKLIST, CONTRIBUTING described v1.2-era preset model after architectural pivot to goal-based routing; user caught manually post-merge; PR #42 fixed) (WATCH — 1/3) | Minor+ version pivots can ship with public-facing artifacts (README, SETUP-CHECKLIST, CONTRIBUTING, GitHub release body, repo description, topics) describing a prior-era architecture. No agent or CI gate detected the drift before merge. First identified in v2.4 when a fundamental architectural pivot (7-preset pick-list to 3-path goal routing) shipped without updating any public docs. Proposed mitigation: Phase 7 checklist should include an explicit public-artifact audit step for minor+ releases — requires attestation that all public-facing docs describe the current architecture. Council self-improve candidate: C1 council-public-artifact-hygiene (HIGH). | 2026-05-09 |
| Verifier-Phrasing-Gap | INFO | v2.4 (first instance — 5 Phase 5 INFO items where constraint verifier grep patterns did not match @dev's idiomatic implementation; all non-blocking) (WATCH — 1/3) | Constraint verifiers authored at Phase 1 against @architect prose idioms (e.g., `skills/<skill-name>/SKILL.md`, `'WIZARD.md Step 4'` literal) do not match @dev's canonical idioms (e.g., `skills/<slug>/SKILL.md`, backtick-formatted step references). Verifiers fire as false-negative INFO — implementation is correct, grep pattern is imprecise. Risk: if a future cycle ships an actual defect in the same location, the imprecise verifier produces false-pass and the defect ships. Proposed mitigation: @architect verifier authoring checklist should include "match @dev's file-path and formatting conventions." Severity INFO because all v2.4 instances were verifier-wrong/implementation-correct; escalate to WARNING if a false-pass allows a defect to reach Phase 7. | 2026-05-09 |
